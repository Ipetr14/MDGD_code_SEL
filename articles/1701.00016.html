Non-Negative Matrix Factorization Test Cases

Connor Sell and Jeremy Kepner
Massachusetts Institute of Technology, Cambridge, MA 02139
email: csell@mit.edu, kepner@ll.mit.edu
Abstract

Non-negative matrix factorization (NMF) is a problem with many applications, ranging from facial recognition to document clustering. However, due to the variety of algorithms that solve NMF, the randomness involved in these algorithms, and the somewhat subjective nature of the problem, there is no clear “correct answer” to any particular NMF problem, and as a result, it can be hard to test new algorithms. This paper suggests some test cases for NMF algorithms derived from matrices with enumerable exact non-negative factorizations and perturbations of these matrices. Three algorithms using widely divergent approaches to NMF all give similar solutions over these test cases, suggesting that these test cases could be used as test cases for implementations of these existing NMF algorithms as well as potentially new NMF algorithms. This paper also describes how the proposed test cases could be used in practice.

I Introduction

What do document clustering, recommender systems, and audio signal processing have in common? All of them are problems that involve finding patterns buried in noisy data. As a result, these three problems are common applications of algorithms that solve non-negative matrix factorization, or NMF [2, 6, 7].

Non-negative matrix factorization involves factoring some matrix 𝐀𝐀\mathbf{A}, usually large and sparse, into two factors 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H}, usually of low rank

𝐀=𝐖𝐇𝐀𝐖𝐇\mathbf{A}=\mathbf{WH} (1)

Because all of the entries in 𝐀𝐀\mathbf{A}, 𝐖𝐖\mathbf{W}, and 𝐇𝐇\mathbf{H} must be non-negative, and because of the imposition of low rank on 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H}, an exact factorization rarely exists. Thus NMF algorithms often seek an approximate factorization, where 𝐖𝐇𝐖𝐇\mathbf{WH} is close to 𝐀𝐀\mathbf{A}. Despite the imprecision, however, the low rank of 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H} forces the solution to describe 𝐀𝐀\mathbf{A} using fewer parameters, which tends to find underlying patterns in 𝐀𝐀\mathbf{A}. These underlying patterns are what make NMF of interest to a wide range of applications.

In the decades since NMF was introduced by Seung and Lee [5], a variety of algorithms have been published that compute NMF [1]. However, the non-deterministic nature of these NMF algorithms make them difficult to test. First, NMF asks for approximations rather than exact solutions, so whether or not an output is correct is somewhat subjective. Although cost functions can quantitatively indicate how close a given solution is to being optimal, most algorithms do not claim to find the globally optimal solution, so whether or not an algorithm gives useful solutions can be ambiguous. Secondly, all of the algorithms produced so far are stochastic algorithms, so running the algorithm on the same input multiple times can give different outputs if they use different random number sequences. Thirdly, the algorithms themselves, though often simple to implement, can have very complex behavior that is difficult to understand. As a result, it can be hard to determine whether a proposed algorithm really “solves” NMF.

This paper proposes some test cases that NMF algorithms should solve verifiably. The approach uses very simple input, such as matrices that have exact non-negative factorizations, that reduce the space of possible solutions and ensure that the algorithm finds correct patterns with little noise. In addition, small perturbations of these simple matrices are also used, to ensure that small variations in the matrix 𝐀𝐀\mathbf{A} do not drastically change the generated solution.

Note that some solutions did not have two negligible entries among 𝐰1​(1)subscript𝐰11\mathbf{w}_{1}(1), 𝐰1​(2)subscript𝐰12\mathbf{w}_{1}(2), 𝐡1​(1)subscript𝐡11\mathbf{h}_{1}(1), and 𝐡1​(2)subscript𝐡12\mathbf{h}_{1}(2), in which case the smaller entry was ignored for the sake of sorting - this accounted for about 20% of the three algorithms, the majority occurring in Algorithm 1.

It is significant to note that even the solutions that didn’t fall cleanly into a “type” still satisfied the pattern given in (12). It seems that an NMF algorithm should satisfy this pattern, but little more is required.

Next, entries in 𝐚0subscript𝐚0\mathbf{a}_{0} and 𝐚1subscript𝐚1\mathbf{a}_{1}, were changed as in the 2×2222\times 2 case. As long as the entries were O​(1)𝑂1O(1) (as opposed to O​(ϵ)𝑂italic-ϵO(\epsilon) or O​(1ϵ)𝑂1italic-ϵO(\frac{1}{\epsilon})), the behavior of the algorithms was similar.

Finally, 𝐀𝐀\mathbf{A} larger than 3×3333\times 3 were examined. Several different sizes of matrices were tested, ranging from 4×4444\times 4 to 20×20202020\times 20, always keeping 𝐀𝐀\mathbf{A}, 𝐖𝐖\mathbf{W}, and 𝐇𝐇\mathbf{H} square, with positive entries only on the main diagonal and the superdiagonal. The experiments followed the same general pattern; nonzero entries in 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H} appeared only on the main diagonal and superdiagonal. Using similar logic to the 2×2222\times 2 and 3×3333\times 3 cases, it can be shown that these are the only exact solutions. However, in practice, as the matrices get larger, exceptions to this pattern become more common, particularly in Algorithm 3. The general rule seems to mostly hold (over half the time) until 𝐀𝐀\mathbf{A} becomes around 20×20202020\times 20. Note, however, that because the run-time of the algorithms are cubic in the size of the matrix, at best, the sample size for large matrices is small.

IV Proposed Tests for NMF Algorithms

Since all three algorithms, which cover a variety of approaches to NMF, had a lot in common in their solutions, it is propose that these inputs 𝐀𝐀\mathbf{A} could be used as a test case of an NMF algorithm implementation. In this section, it is proposed how such test cases could be executed.

The test begins with input of the form

𝐀=𝐀0+ϵ​𝐀1=diag​(𝐚0)+ϵ​diag​(𝐚1,1)𝐀subscript𝐀0italic-ϵsubscript𝐀1diagsubscript𝐚0italic-ϵdiagsubscript𝐚11\mathbf{A}=\mathbf{A}_{0}+\epsilon\mathbf{A}_{1}=\text{diag}(\mathbf{a}_{0})+\epsilon\text{diag}(\mathbf{a}_{1},1) (13)

𝐀𝐀\mathbf{A} is square, and preferably somewhere between 3×3333\times 3 and 8×8888\times 8 in size, although bigger inputs may be useful as well. The entries should vary between tests. Each test should start by using ϵ=0italic-ϵ0\epsilon=0 so that 𝐀𝐀\mathbf{A} is diagonal. The results of this test should have 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H} monomial - only one nonzero element in each row and column. Ignore entries that are below O​(10−10)𝑂superscript1010O(10^{-10}), for the entirety of testing, as any such entries are negligible.

If 𝐖𝐖\mathbf{W} or 𝐇𝐇\mathbf{H} is not monomial, or if the product 𝐖𝐇𝐖𝐇\mathbf{WH} is not equal to 𝐀𝐀\mathbf{A} to within a negligible margin of error, the algorithm fails the test. Otherwise, the generated solution can be used to find the permutation matrix 𝐏𝐏\mathbf{P} that makes 𝐖𝐏𝐖𝐏\mathbf{WP} and 𝐏−1​𝐇superscript𝐏1𝐇\mathbf{P}^{-1}\mathbf{H} diagonal by replacing the nonzero entries of 𝐇𝐇\mathbf{H} with 1’s. Since 𝐀=𝐖𝐇𝐀𝐖𝐇\mathbf{A}=\mathbf{WH} is diagonal, 𝐖𝐏𝐖𝐏\mathbf{WP} is also diagonal, and since 𝐈=𝐏−1​𝐏𝐈superscript𝐏1𝐏\mathbf{I}=\mathbf{P}^{-1}\mathbf{P} is diagonal, so is 𝐏−1​𝐇superscript𝐏1𝐇\mathbf{P}^{-1}\mathbf{H}. Knowing 𝐏𝐏\mathbf{P} will make the rest of the testing much simpler since it is easier to identify whether a solution is of the form given above when it is not permuted.

Next, run the test again using a positive value for ϵitalic-ϵ\epsilon; ϵ=10−3italic-ϵsuperscript103\epsilon=10^{-3} seems to work well, although using a variety of ϵitalic-ϵ\epsilon is also recommended. Make sure to use the same random seeds that were used in the ϵ=0italic-ϵ0\epsilon=0 test to produce corresponding output. Then check that the 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H} given by the algorithm are such that 𝐖𝐏𝐖𝐏\mathbf{WP} and 𝐏−1​𝐇superscript𝐏1𝐇\mathbf{P}^{-1}\mathbf{H} have nonzero entries only on the two diagonals that they are supposed to. If this doesn’t hold, changing ϵitalic-ϵ\epsilon might have changed which permutation returns 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H} to the proper form, so check again; this happens more commonly among larger matrices than smaller ones. However, if 𝐖𝐖\mathbf{W} and 𝐇𝐇\mathbf{H} really do break the form, or 𝐀≠𝐖𝐇𝐀𝐖𝐇\mathbf{A}\neq\mathbf{WH}, the algorithm fails the test on this input. Otherwise, it passes.

Note that even widely accepted algorithms do fail these tests occasionally, especially with matrices larger than 8×8888\times 8, so it’s advisable to perform the test many times to get a more accurate idea of an algorithm’s performance.

V Conclusion

This paper proposes an approach to the problem of testing NMF algorithms by running the algorithms on simple input that can produce an exact non-negative factorization, and perturbations of such input. In particular, square matrices with O​(1)𝑂1O(1) entries on the main diagonal and O​(ϵ)𝑂italic-ϵO(\epsilon) entries on the superdiagonal are proposed, because they have exact solutions that can enumerated mathematically, or because they are perturbations of matrices with exact solutions.

The test cases have been used as input on three known NMF algorithms that represent a variety of algorithms, and all of them behaved similarly, which suggests testable, quantifiable behaviors that many NMF algorithms share. These test cases offer one approach for testing candidate NMF implentations to help determine whether it behaves as it should.
